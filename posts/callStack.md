## call stack(호출 스택)
자바스크립트의 원리와 개념 그리고 동작이 어떻게 되는지 등을 공부하면서
들어봤을만한 용어다.

‘자바스크립트는 단일 호출 스택으로 한 개의 콜 스택(호출 스택)을 사용한다’라는 것은 흔히 들어봤을 문장이다.


또한 구글에 검색을 했을 때도 대부분 비슷한 설명을 읽기 쉽게 포스팅해놓은 자료 또한 무수히 많을 것이다.


#
간단하게 스택에 대해 짧게 설명을 한자면 스택은 자료구조에서 흔히 봤던 큐(queue)와 스택(stack)에서 그 ‘스택’이다.

스택은 LIFO(last in, first out) 방식으로 마지막에 들어온 녀석이 가장 빨리 삭제(pop) 된다.


#
원통형 과자 프링글스는 가장 아래에 있는 한 조각이 제일 먼저 원통에 들어왔지만, 가장 마지막에 쌓였을 한 조각을 먼저 먹게 된다. 위에서부터 순차적으로 먹기 때문에 이 또한 스택이라 할 수 있다.


#
다시 본론으로 돌아와서 함수(function)를 호출(call) 할 때마다 스택에 순차적으로 쌓인다



```javascript
function one(){
	two();
}
function two(){
	three()
}
function three(){
	console.log("i'm three")
}

one();
```

위와 같이 함수 one, two, three를 선언 후 one을 호출했을때 순차적으로
one이 실행되고 one함수 내부에서 two를 호출하고 two에서는 three를 호출한다.


이때 호출스택에는 호출 순서대로 one, two, three 순으로 쌓이게 되고, 자기 역할이 끝난 함수는 스택에서 또한 삭제 될 것이다.

#
## stack overflow
위에 정상적인 흐름대로라면 스택에 쌓이고, 제거되는 흐름이 반복이 되어야 정상이다.


하지만 의도치 않은 함수의 무한 반복 예시로 연속된 재귀 호출 등으로 인해 함수의 실행이 끝나지 않아 스택에서 사라지지 않고 쌓이기만 하면 스택의 사이즈를 초과하여 오류가 발생한다.


#
이를 ‘stack overflow’라고 한다.


간단하게 stack overflow를 구현한 재귀 함수 예시는 아래와 같다

``` javascript
function printNumber(number) {
  console.log(number);
  printNumber(number + 1);
}
printNumber(1);
```

함수 내에서 자기 자신을 호출하는 흔히 말하는 ‘재귀 함수’로 인해 실행된 함수는 스택에 쌓이기만 하고 삭제되지 않는다.

그러면서 무수히 많은 숫자를 출력하다가 결국에는
*Maximum call stack size exceeded* 라는 오류가 발생한다.
